# APIProbe - Visual Branding Prompts

## Brand Overview

**Tool Name:** APIProbe  
**Tagline:** Validate Before You Devastate  
**Purpose:** API Configuration Validator for AI providers  
**Visual Theme:** Diagnostic/medical scanner meets detective magnifying glass  
**Color Palette:** Beacon HQ Visual System v1

---

## 1. TITLE CARD (16:9 Landscape - 1920x1080px)

**For:** GitHub repository header image, presentations, documentation

**DALL-E Prompt:**
```
Create a sleek, modern title card for "APIProbe" - an API configuration validator tool.

STYLE:
- Dark navy blue background (#0a0e27)
- Bright cyan accents (#00d4ff)
- Tech-forward, professional aesthetic
- Minimalist geometric patterns

LAYOUT:
- Large bold title "APIProbe" centered-left
- Smaller tagline below: "Validate Before You Devastate"
- Visual element: A stylized magnifying glass with scan lines examining 
  floating API endpoint bubbles (showing provider icons: Google, Anthropic, OpenAI)
- Subtle circuit board patterns connecting the API bubbles
- A green checkmark on some bubbles, red X on one (representing validation)

VISUAL METAPHOR:
- Magnifying glass = deep inspection
- Scan lines = diagnostic analysis
- Green/red indicators = pass/fail validation

MOOD: Professional, diagnostic, precise, reliable, Team Brain ecosystem
```

**For:** Desktop app icon, mobile app icon, favicon

```
Create a simplified app icon for "APIProbe" tool.

REQUIREMENTS:
```

## Color Specifications

**Primary Colors:**
- Dark Navy: `#0a0e27` (background)
- Bright Cyan: `#00d4ff` (primary accent)

**Secondary Colors:**
- Deep Purple: `#1a0d2e` (gradients)
- Success Green: `#00ff88` (validation pass)
- Error Red: `#ff4466` (validation fail)
- Warning Amber: `#ffaa00` (warnings)

- White: `#ffffff` (primary text)
- Light Gray: `#e0e0e0` (secondary text)

**Recommended Fonts:**
- Titles: Roboto Bold, Inter Bold, or SF Pro Display

## Visual Metaphors

The APIProbe brand should evoke:

1. **Magnifying Glass** - Deep inspection, finding hidden issues
2. **Medical Probe** - Diagnostic, precise, clinical analysis
3. **Scanner** - Systematic checking, validation
4. **Checkpoint** - Pass/fail gates, quality assurance

Avoid:
- Generic search icons
- Overly complex illustrations
- Multiple competing focal points
- Gradients that reduce contrast

---

## Usage Examples

### GitHub Header
The title card should be used as the repository header image, providing immediate brand recognition and conveying the tool's purpose.

### Documentation
The horizontal logo works well at the top of README files and documentation pages, establishing brand consistency across Team Brain tools.

### Favicon/App Icon
The simplified app icon should be used for browser tabs, desktop shortcuts, and any small-format applications.
